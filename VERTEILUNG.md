# Verteilung der Lernreise

## Empfohlener Weg

Für einen Test mit Kolleginnen und Kollegen ist ein gehosteter Link am angenehmsten.

Empfohlen:

1. Projekt nach GitHub pushen
2. mit Vercel verbinden
3. Deployment Protection aktivieren
4. geschützten Vorschau-Link teilen

## Lokaler Test auf einem Windows-Rechner

### Einfache Startdatei

- `Lernreise-starten.bat`

## Hinweise

- Fortschritt wird lokal im Browser gespeichert
- unterschiedliche Testpersonen sollten möglichst unterschiedliche Browserprofile oder Geräte verwenden
- für gemeinsame Remote-Tests ist Hosting deutlich robuster als ein lokaler Rechner
- für GitHub und Vercel siehe auch `README.md` und `DEPLOYMENT.md`
